# Submission Checklist

This file tracks the exact deliverables requested in the Vanco assessment brief.

## Repository-Level Deliverables

- [ ] One GitHub repository or compressed folder containing all three use cases
- [ ] Root README explaining how to run each solution
- [ ] Architecture diagrams for all three use cases
- [ ] Short report or slides summarizing approach, results, limitations, and next steps
- [ ] External help, pretrained models, tutorials, public code, and AI assistance disclosed

## Use Case 1: Grocery Sales Forecasting

- [ ] Complete training and inference pipeline
- [ ] Time-aware validation/backtesting strategy
- [ ] External/event features included
- [ ] Baseline model implemented
- [ ] Main model implemented
- [ ] Kaggle submission file generated
- [ ] Kaggle leaderboard screenshot included with username and score
- [ ] Architecture or pipeline diagram included
- [ ] Feature importance or explainability analysis included
- [ ] Error analysis by store, product family, holiday, and promotion periods
- [ ] Short limitations and improvement plan included

## Use Case 2: ASL Detection

- [ ] At least 8 ASL classes collected
- [ ] Minimum 20 images per class
- [ ] Bounding box annotations in YOLO, COCO, or Pascal VOC format
- [ ] Dataset summary included
- [ ] Annotation samples included
- [ ] Object detection model trained
- [ ] Data augmentations documented
- [ ] mAP, precision, recall, confusion matrix, and per-class performance reported
- [ ] Webcam demo script included
- [ ] Live demo displays bounding box, predicted sign, confidence score, and latency/FPS
- [ ] Backup recorded demo prepared
- [ ] Deployment constraints discussed

## Use Case 3: Hybrid RAG Application

- [ ] NCERT Class 12 Physics Part 1 PDF ingested
- [ ] Page numbers, sections, formulas, tables, and metadata preserved where possible
- [ ] Chunking strategy documented
- [ ] Vector database/index implemented
- [ ] Graph database or graph representation implemented
- [ ] Keyword search implemented
- [ ] Semantic search implemented
- [ ] Hybrid retrieval combines semantic, keyword, and graph retrieval
- [ ] Answer generation grounded only in retrieved evidence
- [ ] Unsupported questions handled safely
- [ ] Citations or page/section references shown
- [ ] Retrieval evidence visible in live demo
- [ ] Example question set and diagnostics included
- [ ] Limitations and improvement plan included

## Final Review Before Submission

- [ ] All commands in README files have been tested
- [ ] Large artifacts are handled using release assets, cloud links, or documented download steps
- [ ] Secrets and API keys are not committed
- [ ] Screenshots/logs are included where required
- [ ] Live demo instructions are clear enough for a reviewer to reproduce
